# Contributing

모든 기여는 소스 코드와 재현 가능한 합성 테스트만으로 구성해야 합니다.

## 제출 전 확인

```powershell
./scripts/Test-SourceBoundary.ps1
./scripts/Build.ps1
```

다음 자료는 링크 대신 파일을 첨부하는 방식도 허용하지 않습니다.

- 게임 클라이언트, PIN, 데이터 아카이브와 추출 리소스
- 패킷 캡처, 메모리/크래시 덤프, 디컴파일 또는 디스어셈블 산출물
- IDA/x64dbg 프로젝트, VM 이미지, 외부 분석 도구
- 실제 계정 DB, 사용자명, 접속 로그, 토큰과 로컬 절대경로

프로토콜 회귀 테스트에는 직접 생성한 최소 byte fixture와 fixture 생성 코드를
함께 사용하세요. 외부 프로젝트에서 코드를 가져오거나 수정할 때는 먼저 명시적
라이선스를 확인하고 `THIRD_PARTY_NOTICES.md`에 출처, 버전, 변경 여부를 기록해야
합니다.

호환성 변경은 다음을 설명해야 합니다.

1. 변경되는 공개 동작과 실패 모드
2. 서버·접속기 양쪽에 미치는 영향
3. 추가하거나 갱신한 합성 테스트
4. 영속 데이터 형식의 마이그레이션 필요 여부
